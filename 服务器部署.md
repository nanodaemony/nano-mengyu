# 服务器部署指南

**服务器地址:** http://101.200.167.175:3001
**项目路径:** `/root/mengyu-master/`

## 部署前须知

### `.gitignore` 关键配置

```gitignore
# 本地数据（不提交到版本库）
data/
```

`data/` 目录下的数据文件（如 `ideas.json`）不会被 git 跟踪，部署后需要手动创建或从本地复制。

### 平台原生绑定

项目使用 `@tailwindcss/oxide`（Tailwind CSS v4 的 Rust 原生模块），该模块通过 optional dependencies 分发各平台二进制文件：

- `@tailwindcss/oxide-linux-x64-gnu` — Linux x64
- `@tailwindcss/oxide-darwin-arm64` — macOS (Apple Silicon)
- `@tailwindcss/oxide-win32-x64-msvc` — Windows x64

**必须在目标服务器上直接运行 `npm install`**，不能从开发机复制 `node_modules`。

## 部署步骤

```bash
# 1. 克隆代码
git clone <仓库地址> /root/mengyu-master
cd /root/mengyu-master

# 2. 安装依赖（在服务器上执行，确保平台绑定正确）
rm -rf node_modules package-lock.json  # 可选，清理旧依赖
npm install

# 3. 创建数据目录（由于 data/ 在 .gitignore 中）
mkdir -p data
# 然后从本地复制 data/ideas.json 到该目录，或手动创建

# 4. 构建生产版本
npm run build

# 5. 启动生产服务器（端口 3001）
npm start -- -p 3001
```

## 生产模式 vs 开发模式

| 模式 | 命令 | 说明 |
|------|------|------|
| 生产模式 | `npm run build` + `npm start -- -p 3001` | **推荐**。预编译，性能好，无实时编译开销 |
| 开发模式 | `npm run dev` | 仅用于本地开发，服务器上不推荐 |

## 常见问题

### 1. `@tailwindcss/oxide` 原生绑定缺失

**错误信息:**
```
Error: Cannot find native binding. npm has a bug related to optional dependencies
```

**原因:** `node_modules` 是从其他平台（如 Windows）复制的，缺少 Linux 原生绑定。

**解决:** 在服务器上重新安装：
```bash
rm -rf node_modules package-lock.json
npm install
```

### 2. `data/ideas.json` 不存在

如果首次部署或从 git clone，`data/` 目录为空。

**解决:** 从本地复制 `data/ideas.json` 到服务器项目目录下的 `data/` 文件夹中，或通过应用页面正常创建新卡片。

### 3. 卡片列表一直转圈加载

**可能原因:**
- 开发模式下 (`npm run dev`) CSS 编译失败导致服务不稳定 → 改为生产模式
- API 路由编译失败导致请求挂起 → 重启服务、检查控制台报错

## 进程管理

目前使用前台运行，建议后续配置 PM2 或 systemd 实现守护进程：

```bash
# PM2 示例
npm install -g pm2
pm2 start npm --name "nano-portal" -- start -- -p 3001
pm2 save
pm2 startup
```
